2.重新注册Gitlab Runner

  1. 停止 GitLab Runner 服务：

    xxxxxxxxxx
    sudo gitlab-runner stop && rm /etc/gitlab-runner/config.toml

  2. 取消注册所有现有的 Runner：

    xxxxxxxxxx
    sudo gitlab-runner unregister --all-runners
    

    rm /etc/gitlab-runner/config.toml

  3. 重新注册 GitLab Runner：

    xxxxxxxxxx
    sudo gitlab-runner register

    按提示操作(以zhaohao为例,wsl和hypeV)：

    • 输入你的 GitLab 实例 URL：

      xxxxxxxxxx
      http192.168.6.94

    • 输入注册 token:

      xxxxxxxxxx
      GR1348941_tNBaqkozChyypeNJxPD

    • 输入此 Runner 的描述: zhaohao1-wsl输入标签: wsl,zhaohao(hv,zhaohao)

    • 选择执行器，对于 WSL，通常选择 shell 或者 docker 执行器最为合适。

  4. 重新启动 GitLab Runner 服务：

    xxxxxxxxxx
    sudo gitlab-runner restart

如果需要特定 Runner 的取消注册，可以使用其 Runner token，例如：

xxxxxxxxxx
sudo gitlab-runner unregister --token YOUR_RUNNER_TOKEN

6.设置ubuntu静态ip(创建hyperV虚拟交换机)

6.1 创建内部虚拟交换机

  1. 打开 PowerShell 以管理员身份运行。

  2. 创建一个内部虚拟交换机：

    xxxxxxxxxx
    New-VMSwitch -SwitchName "internal" -SwitchType Internal

6.2 配置内部交换机的 IP 地址

  1. 获取新创建的虚拟交换机的接口索引：

    xxxxxxxxxx
    Get-NetAdapter -Name "vEthernet (internal)"

    记下 ifIndex 值。

  2. 配置 IP 地址：

    xxxxxxxxxx
    New-NetIPAddress -IPAddress 192.168.2.1 -PrefixLength 24 -InterfaceIndex <ifIndex>

6.3 配置虚拟机的网络设置

xxxxxxxxxx
sudo vim /etc/netplan/50-cloud-init.yaml
xxxxxxxxxx
network:
  version: 2
  renderer: networkd
  ethernets:
    eth0:
      dhcp4: yes
    eth1:
      dhcp4: no
      addresses:
        - 192.168.2.3/24
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4
